7. Clicando na parte inferior esquerdo, teremos um botando servindo para logar a conta do gitHub com o VScode.
8. O repositório vem servindo como uma parta que guarda documentos e até mesmo arquivos diretórios, com históricos de modificações e dados associados.
9. As configurações irão ser passadas na hora que for feito o commit, servindo para alterar as variáveis dos dados dos usuários. 
10. Criação do repositório. Ao digitar GIT INIT . Este comando irá criar um repositório local na pasta em que você está.
11. No comando em verde NEW , você consegue adicionar as informações do repositório em CREATE REPOSITORY.
12. O staging é uma área intermediária, que será usada para criar commit.
13. O comando, git rm --cached <nome do arquivo> você estará removendo o arquivo do staging. 
14. Já a criação do commit seria, git commit 'm "mensagem do commit" . Após o comando as mudanças ficara no repositório local
15. Para enviar uma nova linha do tempo para o repositório remoto seria, git remote add origin <url> 
16. Conectando o repositório, é  necessário ter cadastro uma chave SSH, no github. 
17. O comando >/ git pull, atualiza o repositório local com o que tem no repositório remoto e o >/ git push atualiza o repositório remoto com o que tem no repositório local.
18. O git clone, é uma forma de clonar toda a linha do tempo.
19. Desfazendo mudanças, como método git reverter HEAD , criando um novo commit revertendo todas as mudanças 
20. A branches, sempre aponta para o commit mais atualizado 
21. O comando git checkout é usado para se mover entre as nações.
